The Flaky Network

Mission Objective: Engineer a C++ transport layer to carry a live, 160-byte 50fps audio stream across a highly volatile UDP relay network. The network features packet drops, heavy jitter, sequence reordering, and duplication. The architecture must achieve a ≤ 1.00% deadline miss rate while strictly remaining under a ≤ 2.00x bandwidth multiplier.

1. Mathematical Rejection of Naive Transport Strategies

Before implementing the C++ engine, we rigorously evaluated standard transport paradigms against the physical constraints of the assignment to prove their inadequacy:

  • Why TCP / NACK Retransmission Fails: A lost packet detected at the receiver requires a Negative Acknowledgment (NACK) to travel back through the hostile relay, and the retransmitted packet to traverse the relay a third time. At 50fps (20ms per frame), this triple round-trip delay mathematically guarantees a deadline miss on volatile network profiles (Profile B). Real-time audio cannot tolerate round-trip penalties.
  • Why RFC-2198 (Payload Piggybacking) Fails: Sending the current frame (160 bytes) and a redundant copy of the previous frame (160 bytes) costs exactly 320 bytes. The 2.0x assignment budget is precisely 328 bytes. The moment we append the required sequence IDs and routing headers, the packet inflates past 328 bytes. This pushes the bandwidth overhead to ~2.03x, which results in an instant, hard-coded INVALID score.

2. Chosen Architecture: Adaptive Interleaved FEC (K=2)

The Solution: A proactive XOR parity engine. For every 2 data frames, the C++ sender computes and emits exactly 1 parity packet (P = Frame_A ⊕ Frame_B).

Because this Forward Error Correction (FEC) operates at a strict 3:2 mathematical ratio, the bandwidth overhead is precisely 1.54x. This perfectly satisfies the ≤ 2.00x bandwidth constraint while leaving a massive bandwidth buffer to accommodate network re-routing and custom telemetry.

Defeating Micro-Bursts via Stride Interleaving

Network drops on hostile relays are rarely isolated; they arrive in consecutive micro-bursts. A naive parity engine (Pairing Frame 0 and Frame 1) fails catastrophically if both frames are dropped in a single burst. We solved this via Mathematical Stride Interleaving.

By setting Stride = 2, Frame 0 pairs algebraically with Frame 2. A 2-packet network burst that completely destroys Frame 1 and Frame 2 segregates those losses into entirely independent parity equations, ensuring 100% mathematical recovery without requiring a retransmission.

3. Closed-Loop Telemetry & Adaptive Jitter Control

While high strides provide ultimate burst protection, they physically demand a larger delay_ms jitter buffer at the receiver to wait for the distant parity partner to arrive. Rather than penalizing clean networks with unnecessary latency, we engineered a closed-loop, low-latency telemetry feedback system.

The C++ Receiver maintains a 500ms sliding window, analyzing the exact network burst-loss pattern in real-time. Every 500ms, it transmits an ultra-lightweight 8-byte telemetry packet back to the sender (costing only 480 bytes/sec). If the network is pristine, the sender dynamically shifts to Stride 1, maximizing available jitter-buffer headroom. If a burst is detected, it instantly shifts to Stride 2 to weather the storm.

4. Low-Level C++ Systems Engineering

The sender and receiver are engineered in pure C++17, designed to operate in a zero-delay, high-throughput environment. The network pipeline utilizes strict fcntl(O_NONBLOCK) socket configurations and highly tuned select() event loops to guarantee packet intake and parity generation are never delayed by OS kernel blocking.

Memory Architecture: Perpetual Ring Buffers & Garbage Collection

To guarantee the endpoints can process infinite real-time streams (e.g., automated stress tests lasting for hours) without memory fragmentation, all internal packet tracking is governed by modulo-indexed static Ring Buffers (seq % 4096).

Because C++ dynamic allocation (new/malloc) induces latency spikes, our main loop is 100% zero-allocation. Furthermore, the receiver implements an Age-Out Sweeper that automatically garbage-collects stale parity packets older than 2 seconds, guaranteeing the recovery engine never exhausts its static memory pools under extreme duress.

Bit-Level Wire Protocol

To explicitly bypass host endianness vulnerabilities (Little vs. Big Endian), the packet headers are manually bit-shifted into Network Byte Order. The protocol is strictly packed to exactly 164 bytes to preserve the 1.54x overhead calculation.
